### Практическое задание: знакомство с Vite

**Цель**
Освоить базовую работу с Vite: создание проекта, запуск dev-сервера, организацию структуры файлов, подключение стилей, работу со статическими ресурсами и сборку проекта для production.

### Задание (шаги)

**Тема работы:** создать небольшой одностраничный проект на Vite — например, страницу-визитку, каталог карточек или мини-приложение “Список задач”.

#### 1. Создать проект

Создайте новый проект через Vite.
Можно использовать шаблон **Vanilla JavaScript**.

Что нужно сделать:

- инициализировать проект;
- установить зависимости;
- запустить локальный сервер разработки;
- убедиться, что стартовая страница открывается в браузере.

#### 2. Очистить шаблон

Удалите лишние демонстрационные элементы, которые создаются по умолчанию.

Нужно оставить:

- базовую HTML-структуру;
- подключение основного JS-файла;
- файл стилей.

#### 3. Подготовить структуру проекта

Организуйте проект так, чтобы в нем были:

- `index.html`
- папка `src`
- основной файл JS
- отдельный файл CSS
- папка для изображений или других статических ресурсов

Желательно сразу сделать структуру аккуратной и понятной.

#### 4. Реализовать интерфейс страницы

Сверстайте страницу с несколькими блоками. Например:

- заголовок;
- краткое описание;
- секция с карточками;
- кнопка действия;
- подвал страницы.

Минимальные требования:

- не менее 3–4 смысловых блоков;
- стилизация через отдельный CSS-файл;
- адаптивность хотя бы на базовом уровне.

#### 5. Добавить интерактивность через JavaScript

Сделайте хотя бы 2 интерактивных элемента. Например:

- кнопка, меняющая текст на странице;
- переключение темы;
- отображение списка карточек из массива;
- добавление нового элемента в DOM;
- фильтрация данных по нажатию.

#### 6. Использовать импорт модулей

Разделите код минимум на 2 JS-модуля.

Например:

- в одном файле хранить данные;
- в другом — функции рендера;
- в основном файле — точку входа.

#### 7. Подключить статические ресурсы

Добавьте в проект хотя бы один ресурс:

- изображение;
- иконку;
- локальный шрифт;
- JSON-данные.

Подключение должно происходить корректно в среде Vite.

#### 8. Проверить работу dev-режима

Измените несколько файлов и убедитесь, что:

- страница обновляется быстро;
- изменения отображаются без полной ручной пересборки;
- ошибок в консоли нет.

#### 9. Выполнить production-сборку

Соберите проект для production.

После этого:

- проверьте, что появилась папка сборки;
- убедитесь, что проект собирается без ошибок;
- при желании запустите предпросмотр production-версии.

### Подсказки по ключевым частям

#### 1. Создание проекта

Обычно работа начинается с команды инициализации Vite, затем выполняется установка зависимостей и запуск dev-сервера.
Важно понимать разницу между:

- **dev-режимом** — для разработки;
- **build** — для итоговой сборки;
- **preview** — для локальной проверки production-версии.

#### 2. Точка входа

В Vite точкой входа часто выступает `index.html`, а не только JS-файл.
Это важное отличие от некоторых других сборщиков.

#### 3. Импорт файлов

В Vite можно импортировать:

- JS-модули;
- CSS;
- изображения;
- JSON.

Например, ресурсы можно подключать через `import`, если они используются внутри JavaScript.

#### 4. Статические файлы

Если файл должен использоваться “как есть” по прямому пути, его часто кладут в `public`.
Если ресурс должен участвовать в обработке сборщиком, обычно его размещают в `src`.

#### 5. Модульность

Не стоит писать весь код в одном файле.
Лучше сразу разделять:

- данные;
- логику;
- рендеринг;
- стили.

#### 6. Интерактивность

Даже простое действие должно быть оформлено аккуратно:

- получить элемент из DOM;
- повесить обработчик события;
- изменить состояние или содержимое страницы;
- проверить, что поведение работает повторно и без ошибок.

### Что проверить перед отправкой (чек-лист)

- проект создан именно на **Vite**;
- приложение запускается в dev-режиме;
- стартовая страница открывается без ошибок;
- структура проекта организована понятно;
- стили вынесены в отдельный файл;
- есть минимум 2 интерактивных действия на JavaScript;
- код разделен хотя бы на 2 модуля;
- подключен хотя бы один статический ресурс;
- production-сборка проходит без ошибок;
- в консоли браузера нет ошибок;
- интерфейс выглядит аккуратно;
- имена файлов и папок понятные и единообразные.

### Советы по улучшению работы

- Добавьте переключение светлой и темной темы.
- Используйте массив объектов и рендер карточек через JavaScript.
- Реализуйте простую фильтрацию или поиск.
- Сделайте более чистую архитектуру: `data.js`, `render.js`, `main.js`.
- Добавьте hover-эффекты и адаптивность под мобильные устройства.
- Оформите проект в едином стиле, а не как набор случайных блоков.
- Уберите дублирование кода в функциях.
- Дайте элементам осмысленные классы и названия.
- Добавьте README с инструкцией запуска.
- Проверьте, чтобы проект был легко читаемым для другого разработчика.
