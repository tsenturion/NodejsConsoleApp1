## Практическое задание: Fullstack приложение (Node.js + PostgreSQL + React)

## Практическое задание

Реализовать полноценное CRUD-приложение “Список пользователей” с разделением на клиентскую и серверную части.

Стек:

- Backend: **Node.js**
- База данных: **PostgreSQL**
- Frontend: **React**
- Стилизация: CSS (или CSS Modules / styled-components на выбор)

## Цель

На практике закрепить:

- построение REST API на Node.js
- работу с PostgreSQL (CRUD операции)
- интеграцию React с API
- управление состоянием данных на клиенте
- базовую архитектуру fullstack-приложения
- стилизацию интерфейса без UI-фреймворков

## Задание (шаги)

### 1. Сервер (Node.js)

Создать backend-приложение со следующими требованиями:

- инициализировать проект Node.js
- подключить Express
- настроить подключение к PostgreSQL
- реализовать таблицу `users`:
  - id (primary key)
  - name (text)
  - email (text)
  - created_at (timestamp)

### 2. API (CRUD)

Реализовать REST API:

- `GET /users` — получить список пользователей
- `GET /users/:id` — получить пользователя по id
- `POST /users` — создать пользователя
- `PUT /users/:id` — обновить пользователя
- `DELETE /users/:id` — удалить пользователя

### 3. React клиент

Создать React-приложение (через Vite):

Функциональность:

- отображение списка пользователей
- форма добавления пользователя
- редактирование пользователя
- удаление пользователя

### 4. Интеграция с API

- использовать fetch или axios
- реализовать загрузку данных при старте приложения
- обновлять список после CRUD операций
- обрабатывать ошибки запросов

### 5. Стилизация

Оформить интерфейс:

- центрированная форма добавления
- таблица или карточки пользователей
- кнопки редактирования и удаления
- hover-эффекты
- визуальное разделение блоков

## Подсказки по ключевым частям

### 1. Подключение PostgreSQL

- использовать `pg` (node-postgres)
- создать pool подключений
- хранить конфигурацию отдельно от кода сервера

### 2. Архитектура backend

Рекомендуемая структура:

- routes — маршруты API
- controllers — логика обработки запросов
- db — подключение к базе
- services — работа с SQL

### 3. React структура

Рекомендуется разделение:

- components/UserList
- components/UserForm
- services/api.js
- App.jsx — управление состоянием

### 4. Управление состоянием

- хранить список пользователей в state
- обновлять через повторный запрос после изменений
- избегать дублирования состояния

### 5. Работа с формами

- использовать controlled inputs
- хранить значения формы в state
- очищать форму после успешного запроса

### 6. Обработка ошибок

- выводить ошибки в интерфейсе
- обрабатывать пустые поля
- учитывать ошибки сети

## Советы по улучшению работы

### 1. Разделяйте слои

Не смешивайте SQL, бизнес-логику и маршруты в одном файле.

### 2. Не храните лишнее состояние в React

Если данные можно получить с сервера — не дублируйте их в нескольких местах.

### 3. Добавьте индикаторы загрузки

Покажите состояние “loading” при запросах.

### 4. Добавьте валидацию

Минимум:

- email содержит `@`
- name не пустой

### 5. Используйте единый API слой

Все запросы должны быть централизованы (например, `api.js`).

### 6. Улучшение архитектуры (по желанию)

- добавить pagination на backend
- добавить поиск пользователей
- добавить фильтрацию
- добавить debounce для поиска

### 7. Работа с ошибками

Не игнорируйте ошибки API — всегда выводите их в UI или лог.
