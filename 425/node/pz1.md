## Практическое задание

**Тема:** Node.js и npm: создание простого консольного приложения

## Цель

Закрепить базовые навыки работы с **Node.js** и **npm** на практике:

- проверить установленное окружение;
- создать простой проект на Node.js;
- понять структуру проекта;
- запустить JavaScript вне браузера;
- инициализировать проект через npm;
- добавить и использовать внешний пакет;
- оформить проект так, как это делают в реальной разработке.

## Задание (шаги)

### 1. Проверить установку Node.js и npm

Откройте терминал и убедитесь, что Node.js и npm установлены и доступны из командной строки.

Что нужно получить:

- версия Node.js;
- версия npm.

### 2. Создать папку проекта

Создайте отдельную папку для практической работы, например:

`node-practice`

Перейдите в эту папку через терминал.

### 3. Инициализировать проект npm

Создайте файл `package.json` с помощью npm.

В результате в проекте должен появиться базовый файл с информацией о проекте:

- имя;
- версия;
- описание;
- точка входа;
- scripts.

### 4. Создать основной файл приложения

Создайте файл `app.js`.

В этом файле нужно:

- вывести в консоль приветствие;
- вывести версию Node.js;
- вывести текущую дату и время;
- вывести путь к текущей папке проекта.

### 5. Добавить собственную структуру проекта

Организуйте проект так, чтобы в нем были:

- `app.js` — основной файл;
- `package.json` — описание проекта;
- `README.md` — краткое описание проекта;
- папка `utils` — для вспомогательных модулей.

В папке `utils` создайте файл, например `message.js`, который экспортирует функцию для вывода или возврата текста приветствия.

Подключите этот модуль в `app.js` и используйте его.

### 6. Использовать встроенный модуль Node.js

В `app.js` подключите один встроенный модуль Node.js, например:

- `os`,
- `path`,
- `fs`.

С его помощью выведите дополнительную полезную информацию, например:

- имя операционной системы;
- домашнюю директорию пользователя;
- имя текущего файла;
- расширение файла.

### 7. Установить внешний npm-пакет

Добавьте в проект один простой пакет из npm, например для красивого вывода текста в консоль.

После установки:

- импортируйте пакет в `app.js`;
- используйте его в работе программы.

Важно: пакет должен реально применяться, а не просто быть установленным.

### 8. Добавить npm-скрипт для запуска

Настройте в `package.json` раздел `scripts`, чтобы проект можно было запускать не только через `node app.js`, но и через npm-команду.

Например, должен быть сценарий запуска приложения.

### 9. Запустить приложение

Запустите программу и проверьте, что в консоли отображаются:

- приветствие;
- данные из собственного модуля;
- информация о Node.js;
- информация из встроенного модуля;
- результат работы внешнего npm-пакета.

### 10. Подготовить проект к сдаче

Перед отправкой проект должен содержать:

- все исходные файлы;
- `package.json`;
- `package-lock.json`;
- краткий `README.md` с инструкцией по запуску.

## Подсказки по ключевым частям

### 1. Проверка Node.js и npm

Команды:

```bash
node -v
npm -v
```

Что делают:

- `node -v` — показывает установленную версию Node.js;
- `npm -v` — показывает установленную версию npm.

### 2. Создание папки и переход в нее

Команды:

```bash
mkdir node-practice
cd node-practice
```

Что делают:

- `mkdir node-practice` — создает папку проекта;
- `cd node-practice` — переходит в эту папку.

### 3. Инициализация npm-проекта

Команда:

```bash
npm init -y
```

Что делает:

- автоматически создает файл `package.json` со значениями по умолчанию.

Если хотите заполнить поля вручную:

```bash
npm init
```

Что делает:

- запускает пошаговое создание `package.json`.

### 4. Запуск программы на Node.js

Команда:

```bash
node app.js
```

Что делает:

- запускает файл `app.js` через Node.js.

### 5. Подключение собственного модуля

Если в `utils/message.js` вы экспортируете функцию, то в `app.js` ее нужно импортировать и использовать.

Идея:

- в одном файле храните логику;
- в другом — подключаете ее через `require` или другой изучаемый способ модульности.

### 6. Установка внешнего пакета

Общая команда установки пакета:

```bash
npm install имя-пакета
```

Пример:

```bash
npm install chalk
```

Что делает:

- скачивает пакет;
- добавляет его в папку `node_modules`;
- записывает зависимость в `package.json`.

Короткая форма:

```bash
npm i chalk
```

### 7. Проверка установленных пакетов

Команда:

```bash
npm list
```

Что делает:

- показывает установленные зависимости проекта.

Иногда удобно использовать:

```bash
npm list --depth=0
```

Что делает:

- показывает только пакеты верхнего уровня, без вложенных зависимостей.

### 8. Добавление npm-скрипта

В `package.json` можно добавить раздел:

```json
"scripts": {
  "start": "node app.js"
}
```

После этого запускать проект можно командой:

```bash
npm start
```

Что делает:

- запускает команду, указанную в скрипте `start`.

Можно добавить и свой скрипт, например:

```json
"scripts": {
  "start": "node app.js",
  "dev": "node app.js"
}
```

Тогда запуск:

```bash
npm run dev
```

Что делает:

- запускает пользовательский скрипт `dev`.

### 9. Повторная установка зависимостей

Если проект открыли на другом компьютере и папки `node_modules` нет, используйте:

```bash
npm install
```

Что делает:

- устанавливает все зависимости из `package.json`.

### 10. Удаление пакета

Если пакет был установлен по ошибке:

```bash
npm uninstall имя-пакета
```

Пример:

```bash
npm uninstall chalk
```

Что делает:

- удаляет пакет из проекта и из списка зависимостей.

### 11. Какие npm-команды вам могут понадобиться в этой работе

Полезный набор команд для всей практики:

```bash
node -v
npm -v
mkdir node-practice
cd node-practice
npm init -y
npm install chalk
npm list --depth=0
node app.js
npm start
npm run dev
npm install
npm uninstall chalk
```

Не все команды обязательны одновременно, но это полный набор, который может понадобиться по ходу выполнения.

## Советы по улучшению работы

1. Сделайте вывод программы понятным и аккуратным: разделите блоки информации заголовками.
2. Добавьте несколько функций в папку `utils`, а не один файл с единственной строкой.
3. Используйте встроенные модули осмысленно, а не формально.
4. В `README.md` опишите:
   - что делает программа;
   - как установить зависимости;
   - как запустить проект.

5. Придумайте небольшой практический смысл программе, например:
   - “Информация о системе”;
   - “Приветствие пользователя”;
   - “Мини-утилита для разработчика”.

6. Проверьте, что код читаемый: понятные имена переменных, аккуратные отступы, логичная структура файлов.
