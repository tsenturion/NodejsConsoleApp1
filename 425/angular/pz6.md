## Практическое задание

### Тема:

**Создание Angular-проекта через Angular CLI и разбор базовой архитектуры: модуль, компонент, шаблон**

### Цель

В этом практическом задании вам нужно не просто запустить новый Angular-проект, а понять, из каких базовых частей он состоит. В ходе работы вы создадите приложение через Angular CLI, изучите его стартовую структуру и реализуете собственный компонент с шаблоном. Также вы разберётесь, какую роль в проекте играют модуль, компонент и HTML-шаблон, и как они связаны между собой.

## Задание (шаги)

### 1. Установите необходимые инструменты

Перед началом работы убедитесь, что у вас установлены:

- Node.js
- npm
- Angular CLI

После этого проверьте версии командой:

```bash
node -v
npm -v
ng version
```

### 2. Создайте новый Angular-проект

С помощью Angular CLI создайте новый проект.

Пример команды:

```bash
ng new my-first-angular-app
```

Во время создания проекта ответьте на вопросы CLI. После этого перейдите в папку проекта:

```bash
cd my-first-angular-app
```

### 3. Запустите проект

Запустите локальный сервер разработки:

```bash
ng serve
```

Откройте приложение в браузере и проверьте, что стартовая страница Angular отображается корректно.

### 4. Изучите структуру проекта

Найдите в проекте основные файлы и папки, которые относятся к архитектуре Angular-приложения. Вам нужно определить:

- где находится главный модуль приложения;
- где расположен главный компонент;
- где хранится шаблон компонента;
- где находится файл стилей компонента;
- где задаётся логика компонента.

### 5. Разберите роль основных частей архитектуры

На основе структуры проекта кратко опишите своими словами:

- что такое **модуль**;
- что такое **компонент**;
- что такое **шаблон**;
- как компонент связан с шаблоном;
- зачем Angular разбивает приложение на такие части.

### 6. Создайте собственный компонент

Создайте новый компонент через Angular CLI. Например:

```bash
ng generate component student-card
```

или краткая форма:

```bash
ng g c student-card
```

После этого проверьте, какие файлы были созданы автоматически.

### 7. Заполните компонент содержимым

В созданном компоненте реализуйте простую карточку, в которой будут:

- заголовок;
- абзац с описанием;
- список из 3–5 пунктов;
- кнопка.

Например, это может быть карточка студента, мини-блок о курсе, описание темы или карточка учебного модуля.

### 8. Подключите компонент в приложение

Добавьте ваш новый компонент в главный шаблон приложения так, чтобы он отобразился на странице после запуска проекта.

### 9. Измените шаблон и логику компонента

Добавьте в компонент:

- одно строковое свойство в TypeScript;
- одно массивное свойство;
- вывод данных в шаблон;
- простую кнопку, которая вызывает метод компонента.

Например, при нажатии кнопки можно менять текст заголовка или показывать сообщение.

### 10. Оформите результат

Сделайте аккуратную структуру кода:

- понятные имена переменных;
- понятное название компонента;
- читаемая разметка;
- простое оформление через CSS.

## Подсказки по ключевым частям

### 1. Angular CLI нужен не только для создания проекта

CLI — это основной инструмент для старта и сопровождения Angular-приложения. Через него удобно:

- создавать проект;
- запускать локальный сервер;
- генерировать компоненты;
- генерировать модули и другие сущности;
- собирать приложение.

То есть Angular CLI помогает соблюдать единый стандарт структуры проекта.

### 2. Модуль — это контейнер для частей приложения

В классическом понимании Angular модуль объединяет связанные части приложения. В нём Angular понимает, какие компоненты относятся к приложению, какие зависимости подключены и что является точкой входа.

Если вы работаете с базовой архитектурой, то вам важно увидеть, что модуль связывает приложение в единую систему.

### 3. Компонент — это основная строительная единица интерфейса

Компонент отвечает за отдельный участок интерфейса. У него обычно есть три части:

- логика в `.ts`;
- шаблон в `.html`;
- стили в `.css` или `.scss`.

То есть компонент — это не только TypeScript-файл, а целый блок, который управляет куском интерфейса.

### 4. Шаблон показывает данные пользователю

HTML-шаблон нужен для отображения данных из компонента. Именно в шаблоне вы выводите текст, списки, кнопки, блоки и связываете интерфейс с логикой.

Например:

- строковые свойства можно вывести в тексте;
- массивы можно показать списком;
- методы можно вызывать из событий, например по нажатию кнопки.

### 5. Генерация через CLI лучше ручного создания файлов

Когда вы создаёте компонент через команду `ng g c`, Angular автоматически создаёт правильную структуру файлов. Это снижает вероятность ошибок и помогает сразу работать в правильной архитектуре.

## Что проверить перед отправкой (чек-лист)

Проверьте, что у вас выполнено следующее:

- проект создан именно через Angular CLI;
- приложение запускается без ошибок;
- вы можете открыть его в браузере;
- вы нашли и изучили главный модуль проекта;
- вы нашли главный компонент проекта;
- вы нашли HTML-шаблон компонента;
- вы создали новый компонент через CLI;
- новый компонент подключён к приложению;
- в компоненте есть логика в TypeScript;
- в шаблоне выводятся данные из компонента;
- в компоненте есть кнопка с обработчиком события;
- код отформатирован и читается аккуратно;
- названия файлов, компонентов и переменных понятны;
- структура проекта не нарушена.

## Советы по улучшению работы

### 1. Не ограничивайтесь только запуском проекта

Важно не просто выполнить команды, а понять, что Angular CLI создаёт автоматически и почему структура именно такая. Постарайтесь осмысленно посмотреть на каждый основной файл.

### 2. Давайте компонентам понятные имена

Название компонента должно отражать его назначение. Например, `student-card`, `course-info`, `lesson-block` звучат намного лучше, чем абстрактные `test1` или `block`.

### 3. Делите ответственность между логикой и шаблоном правильно

Не нужно пытаться всю логику размещать в HTML. Шаблон должен отображать данные, а основная логика должна находиться в TypeScript-файле компонента.

### 4. Делайте шаблон аккуратным

Даже в учебной работе полезно соблюдать порядок:

- использовать отступы;
- не писать всё в одну строку;
- логично группировать элементы.

### 5. Старайтесь понимать архитектуру, а не заучивать названия

Вам важно увидеть связь:

- модуль собирает части приложения;
- компонент управляет участком интерфейса;
- шаблон отображает данные;
- стили оформляют внешний вид.

Когда вы понимаете эту связь, Angular становится намного проще.

### 6. Попробуйте сделать чуть больше минимума

Например, вы можете:

- добавить второй компонент;
- передать данные в шаблон разными способами;
- оформить карточку красивее;
- сделать два блока на странице вместо одного.
