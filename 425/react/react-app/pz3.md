## Практическое задание

**Тема:** Создание интерактивного списка задач на React

### Цель

Закрепить навыки работы с:

- `props`
- `state`
- обработкой событий
- условным рендерингом

В результате вы должны собрать небольшое React-приложение, в котором можно добавлять задачи, отмечать их как выполненные, удалять и фильтровать отображение.

## Задание (шаги)

### 1. Создайте компонент приложения

Создайте корневой компонент `App`, который будет хранить список задач и текущее состояние интерфейса.

Что должно быть в состоянии:

- массив задач
- значение поля ввода
- выбранный фильтр, например: `all`, `active`, `completed`

Пример структуры одной задачи:

- `id`
- `text`
- `completed`

### 2. Создайте форму добавления задачи

Добавьте:

- поле ввода текста
- кнопку «Добавить»

Требования:

- при вводе текст должен сохраняться в `state`
- при нажатии на кнопку задача добавляется в список
- пустую задачу добавлять нельзя
- после добавления поле очищается

Здесь нужно применить:

- `state` для хранения текста
- обработчик `onChange`
- обработчик `onClick` или `onSubmit`

### 3. Вынесите отображение задачи в отдельный компонент

Создайте компонент, например `TaskItem`.

Он должен получать через `props`:

- текст задачи
- статус выполнения
- функцию переключения статуса
- функцию удаления

Это нужно для практики передачи данных и функций через `props`.

### 4. Реализуйте отметку задачи как выполненной

У каждой задачи должна быть кнопка или чекбокс:

- если задача не выполнена — отметить как выполненную
- если выполнена — снять отметку

При изменении статуса внешний вид задачи должен меняться, например:

- выполненная задача зачёркнута
- рядом выводится текст `Выполнено`

Здесь нужно использовать:

- изменение `state`
- обработку событий
- условный рендеринг

### 5. Реализуйте удаление задачи

У каждой задачи должна быть кнопка «Удалить».

При нажатии задача удаляется из списка.

### 6. Добавьте условный рендеринг для пустого списка

Если задач нет, вместо списка должно отображаться сообщение:
**«Список задач пуст»**

Если задачи есть — отображается список.

### 7. Добавьте фильтрацию задач

Сделайте три кнопки:

- Все
- Активные
- Выполненные

По нажатию должен меняться список отображаемых задач:

- `Все` — показываются все задачи
- `Активные` — только невыполненные
- `Выполненные` — только выполненные

Это позволит дополнительно потренировать:

- `state`
- обработчики событий
- условный рендеринг

### 8. Добавьте счётчик

Выведите информацию:

- общее количество задач
- количество выполненных задач
- количество оставшихся задач

## Подсказки по ключевым частям

### 1. Как использовать props

`props` нужны для передачи данных из родительского компонента в дочерний.

Например:

- `App` хранит список задач
- `TaskItem` получает одну задачу и функции для работы с ней

Полезно помнить:

- через `props` можно передавать не только значения, но и функции
- дочерний компонент не должен напрямую менять состояние родителя

### 2. Как работать со state

`state` нужен для данных, которые меняются во время работы приложения.

В этом задании в `state` удобно хранить:

- список задач
- текущий текст в поле ввода
- активный фильтр

Важно:

- не изменяйте массив задач напрямую
- создавайте новый массив через `map`, `filter`, `spread`

### 3. Обработка событий

Используйте события:

- `onChange` для поля ввода
- `onClick` для кнопок
- `onSubmit`, если используете форму

Примеры действий:

- добавить задачу
- удалить задачу
- переключить статус задачи
- сменить фильтр

### 4. Условный рендеринг

Используйте условный рендеринг для случаев:

- список пуст
- задача выполнена
- показывается определённый тип задач по фильтру

Подходящие варианты:

- тернарный оператор
- логический `&&`
- заранее вычисленная переменная с результатом фильтрации

### 5. Как организовать компоненты

Минимальная структура может быть такой:

- `App`
- `TaskInput`
- `TaskList`
- `TaskItem`
- `FilterPanel`

Это не строгое требование, но так код будет чище и понятнее.

## Что проверить перед отправкой (чек-лист)

Проверьте, что:

- задачи добавляются корректно
- пустая строка не добавляется в список
- после добавления поле ввода очищается
- каждая задача отображается через отдельный компонент
- данные передаются через `props`
- статус задачи можно менять
- выполненная задача визуально отличается от активной
- задачу можно удалить
- при пустом списке показывается сообщение
- фильтры работают правильно
- счётчики отображают корректные значения
- приложение не падает при кликах и вводе
- код разбит на логичные компоненты
- названия переменных и функций понятны

## Советы по улучшению работы

### Базовые улучшения

- добавьте отправку по клавише `Enter`
- добавьте подтверждение перед удалением задачи
- сделайте кнопку очистки всех выполненных задач

### Улучшения интерфейса

- выделите активный фильтр стилем
- добавьте разные стили для выполненных и невыполненных задач
- сделайте адаптивную вёрстку

### Улучшения логики

- сохраняйте задачи в `localStorage`, чтобы они не пропадали после перезагрузки
- сортируйте задачи: сначала активные, потом выполненные
- покажите сообщение, если пользователь пытается добавить пустую задачу

### Улучшения архитектуры

- вынесите логику фильтрации в отдельную функцию
- используйте понятные имена обработчиков: `handleAddTask`, `handleDeleteTask`, `handleToggleTask`
- не дублируйте код в нескольких местах

## Дополнительное условие

Реализуйте редактирование текста задачи:

- по нажатию на кнопку «Редактировать» задача переходит в режим редактирования
- появляется поле ввода
- можно сохранить изменения или отменить редактирование

## Итоговый результат

В конце должно получиться небольшое React-приложение «Список задач», в котором пользователь может:

- добавлять задачи
- отмечать их выполненными
- удалять
- фильтровать
- видеть состояние списка и счётчики
