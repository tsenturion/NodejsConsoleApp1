# Практическое задание

**Тема:** Маршрутизация + управление состоянием в React

## Цель

- Освоить базовую и вложенную маршрутизацию с помощью React Router
- Научиться управлять глобальным состоянием (через Context API или state management)
- Связать состояние приложения с URL (query params, параметры маршрута)
- Разделить ответственность между компонентами, роутами и состоянием
- Реализовать предсказуемую навигацию и синхронизацию UI

## Задание (шаги)

### 1. Создать структуру приложения

Создайте React-приложение с тремя основными страницами:

- `/` — Главная страница
- `/products` — Список товаров
- `/products/:id` — Страница одного товара
- `/cart` — Корзина

Добавьте layout-компонент (шапка + навигация), который используется на всех страницах.

### 2. Настроить маршрутизацию

- Подключите React Router
- Используйте:
  - `BrowserRouter`
  - `Routes`
  - `Route`

- Реализуйте вложенные маршруты (например, layout → страницы)

### 3. Реализовать глобальное состояние

Создайте глобальное состояние для корзины:

**Структура:**

- список товаров в корзине
- количество каждого товара
- общая сумма (можно вычислять)

Используйте один из вариантов:

- `useContext` + `useReducer` (рекомендуется)
- либо любой другой подход

### 4. Страница списка товаров (`/products`)

- Отобразите список товаров (можно захардкодить массив)
- Для каждого товара:
  - название
  - цена
  - кнопка "Добавить в корзину"
  - ссылка на страницу товара

### 5. Страница товара (`/products/:id`)

- Получите `id` из URL (`useParams`)
- Найдите товар по id
- Отобразите подробную информацию
- Добавьте кнопку "Добавить в корзину"

### 6. Страница корзины (`/cart`)

- Отобразите:
  - список добавленных товаров
  - количество
  - итоговую сумму

- Реализуйте:
  - удаление товара
  - изменение количества

### 7. Связь состояния и маршрутов

Добавьте фильтрацию товаров через query-параметры:

Пример:

```
/products?category=electronics
```

- Используйте `useSearchParams`
- Фильтруйте список товаров по параметру

### 8. Обработка несуществующих маршрутов

- Добавьте страницу 404 (`*`)
- Отобразите сообщение "Страница не найдена"

## Подсказки по ключевым частям

### Маршрутизация

- Layout лучше вынести в отдельный компонент с `<Outlet />`
- Не дублируйте layout на каждой странице

### Глобальное состояние

- `useReducer` лучше подходит, чем `useState`, если:
  - есть сложная логика
  - несколько типов действий (add/remove/update)

Пример действий:

- `ADD_TO_CART`
- `REMOVE_FROM_CART`
- `UPDATE_QUANTITY`

### Работа с URL

- URL — это часть состояния приложения
- Не храните фильтры только в state — дублируйте в query params

### Структура проекта

Разделите код:

- `pages/` — страницы
- `components/` — UI-компоненты
- `context/` — глобальное состояние
- `router/` — конфигурация маршрутов

### Идентификаторы

- Используйте стабильные `id`
- Не используйте индекс массива как ключ

## Что проверить перед отправкой (чек-лист)

- [ ] Все маршруты работают без перезагрузки страницы
- [ ] Переходы через `<Link>` / `<NavLink>`, а не `<a>`
- [ ] Корзина сохраняет состояние при переходе между страницами
- [ ] Добавление товара работает из разных страниц
- [ ] Страница `/products/:id` корректно получает `id`
- [ ] При обновлении страницы состояние маршрута не ломается
- [ ] Query-параметры реально влияют на отображение
- [ ] Нет ошибок в консоли
- [ ] Есть страница 404
- [ ] Нет дублирования логики

## Советы по улучшению работы

### 1. Декомпозиция

Не делайте «один огромный компонент».
Разделяйте:

- контейнер (логика)
- презентационный компонент (UI)

### 2. Предсказуемость состояния

Состояние должно изменяться только через действия (actions), а не напрямую.

### 3. Избегайте дублирования

Если логика добавления в корзину используется в нескольких местах — вынесите её.

### 4. UX детали

- Показывайте количество товаров в корзине в header
- Добавьте disable для кнопки, если товар уже добавлен

### 5. Масштабируемость

Подумайте:

- что будет, если товаров станет 1000+
- если данные будут приходить с сервера

### 6. Следующий шаг

После выполнения попробуйте заменить Context на:

- Redux Toolkit
  или
- Zustand

и сравнить архитектуру.
