## Практическое задание

**Тема:** списки, `key` и формы в React

Создайте небольшое приложение **«Список задач»** или **«Список покупок»**, в котором пользователь сможет добавлять элементы через форму, видеть их в списке и удалять ненужные позиции. Главная цель этого задания — не просто вывести массив на экран, а правильно связать между собой три важные темы React: работу со списками, использование `key` и обработку форм.

## Цель

В этом задании вы должны отработать несколько базовых, но очень важных навыков React.

Во-первых, вы научитесь хранить коллекцию данных в состоянии компонента и отображать её через `map()`.
Во-вторых, вы закрепите понимание того, зачем React нужен атрибут `key` при рендеринге списков.
В-третьих, вы потренируетесь создавать управляемую форму, в которой значение поля связано с состоянием через `useState`.
И, наконец, вы соберёте всё это в одно маленькое, но уже вполне практическое приложение.

## Задание (шаги)

### 1. Создайте компонент приложения

Создайте компонент `App`, в котором будет находиться всё приложение.
Подготовьте в нём состояние для списка элементов. На первом этапе можно сделать массив из нескольких объектов, например:

- `id`
- `title`

Например, это могут быть задачи или товары.

Важно, чтобы каждый элемент списка был не просто строкой, а **объектом с уникальным идентификатором**.

### 2. Выведите список на экран

Используйте метод `map()`, чтобы отрисовать все элементы массива в JSX.
Каждый элемент должен отображаться как отдельная строка, карточка или `<li>` внутри `<ul>`.

У каждого элемента списка обязательно должен быть атрибут `key`.

На этом этапе вы должны показать, что понимаете следующее:

- список в React обычно рендерится через `map()`;
- `key` ставится на корневой JSX-элемент, который возвращается внутри `map()`;
- в качестве `key` лучше использовать уникальный `id`, а не индекс массива.

### 3. Добавьте форму

Под списком или над списком создайте форму с полем ввода и кнопкой добавления.
Пользователь должен вводить название новой задачи или нового товара.

Здесь нужно сделать **управляемый input**:

- создать отдельное состояние для текста поля;
- связать `value` у `input` с этим состоянием;
- обновлять состояние через `onChange`.

То есть поле должно полностью управляться React.

### 4. Реализуйте добавление нового элемента

Когда пользователь отправляет форму, новый элемент должен добавляться в массив списка.

Для этого нужно:

- перехватить отправку формы через `onSubmit`;
- отменить стандартное поведение браузера через `event.preventDefault()`;
- проверить, что строка не пустая;
- создать новый объект;
- добавить его в массив состояния.

У нового объекта должен быть уникальный `id`.
После добавления поле ввода нужно очистить.

### 5. Реализуйте удаление элемента

У каждого элемента списка должна быть кнопка, например **«Удалить»**.

При нажатии на кнопку конкретный элемент должен удаляться из массива.
Для этого используйте `filter()`.

Здесь важно, чтобы удалялся именно тот элемент, по которому нажали, а не любой другой.

### 6. Добавьте сообщение при пустом списке

Если список пуст, вместо элементов выведите текст вроде:

`Список пуст`

Это нужно, чтобы потренироваться делать условный рендеринг вместе со списками.

### 7. Оформите результат аккуратно

Сделайте простую, но понятную структуру интерфейса:

- заголовок приложения;
- форма добавления;
- список элементов;
- кнопки удаления;
- сообщение о пустом списке.

Сильный дизайн здесь не требуется, но приложение должно выглядеть логично и читабельно.

## Подсказки по ключевым частям

### Как хранить список

Лучше сразу хранить не массив строк, а массив объектов.
Это нужно потому, что объект удобнее расширять. Сегодня у вас только `title`, а завтра могут появиться `done`, `date`, `category` и так далее.

Пример структуры одного элемента:

```jsx
{
  id: 1,
  title: "Купить молоко"
}
```

### Как выводить список

Обычно это выглядит так: вы берёте массив и внутри JSX проходите по нему через `map()`.

Важно понимать: `map()` возвращает новый массив JSX-элементов, который React потом отображает.

Пример логики:

```jsx
items.map((item) => <li key={item.id}>{item.title}</li>);
```

Главная мысль тут такая: `key` нужен React не для красоты и не для пользователя, а для внутреннего понимания, какой элемент списка является каким при обновлении интерфейса.

### Почему не стоит использовать индекс как `key`

Индекс массива может подойти только в очень простых случаях, когда список никогда не меняется.
Но в этом задании элементы будут добавляться и удаляться. Значит, индексы будут сдвигаться, и React может начать путать элементы между собой.

Поэтому правильнее использовать стабильный уникальный идентификатор.

### Как сделать управляемую форму

Управляемая форма — это форма, в которой значение поля хранится в состоянии React.

Это означает, что:

- `value={text}`
- `onChange={(e) => setText(e.target.value)}`

То есть источник истины здесь — не DOM, а состояние компонента.

### Как добавить новый элемент

При отправке формы нужно создать новый объект и добавить его в массив через разворот старого массива.

Пример идеи:

```jsx
setItems([...items, newItem]);
```

Ещё лучше использовать функциональное обновление состояния:

```jsx
setItems((prevItems) => [...prevItems, newItem]);
```

Так надёжнее, когда новое состояние зависит от предыдущего.

### Как удалить элемент

Удаление обычно делается через `filter()`:

```jsx
setItems((prevItems) => prevItems.filter((item) => item.id !== id));
```

Здесь сохраняются все элементы, кроме того, чей `id` совпал с переданным.

### Как генерировать `id`

Для учебного задания можно использовать:

- `Date.now()`
- `new Date().getTime()`
- отдельный счётчик
- `crypto.randomUUID()` — если среда поддерживает

Главное, чтобы идентификатор был уникальным.

## Что проверить перед отправкой (чек-лист)

- Список хранится в `useState`.
- Каждый элемент списка выводится через `map()`.
- У каждого элемента есть корректный `key`.
- В качестве `key` используется уникальный `id`, а не индекс массива.
- Поле ввода связано с состоянием через `value` и `onChange`.
- Форма отправляется через `onSubmit`.
- При отправке формы страница не перезагружается.
- Пустые строки не добавляются в список.
- После добавления нового элемента поле очищается.
- У каждого элемента есть кнопка удаления.
- Удаление работает корректно.
- При пустом массиве показывается сообщение, что список пуст.
- Код разбит логически и читается без путаницы.

## Советы по улучшению работы

Когда базовая версия будет готова, можно постепенно усложнить приложение.

Сначала можно добавить отметку **«выполнено»** или **«куплено»**, чтобы у каждого элемента было ещё одно поле в объекте. Тогда вы начнёте работать не просто со списком, а со списком объектов с несколькими свойствами.

Потом можно сделать фильтрацию: показывать все элементы, только выполненные или только невыполненные. Это даст хорошую практику по условному рендерингу и работе с массивами.

Ещё полезно добавить счётчик, который показывает, сколько всего элементов в списке. Это простая доработка, но она хорошо тренирует связь между состоянием и интерфейсом.

Можно также вынести один элемент списка в отдельный компонент, например `ListItem`. Тогда вы дополнительно потренируетесь передавать данные через `props`. В таком случае особенно важно не потерять правильное место для `key`: он должен оставаться там, где идёт перебор через `map()`.

Наконец, полезно обратить внимание не только на то, чтобы код работал, но и на то, чтобы он был аккуратно организован. Например, обработчик добавления лучше вынести в отдельную функцию, обработчик удаления — тоже. Тогда JSX будет чище, а логика понятнее.
