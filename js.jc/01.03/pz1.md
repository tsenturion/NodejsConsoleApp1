## Практическое задание

### Тема: «Менеджер списка покупок (консольное приложение)»

## Цель

Закрепить основы JavaScript: базовый синтаксис, условия и циклы, функции (включая callback), массивы и объекты (с методами и деструктуризацией) на практике — через создание небольшого приложения, которое управляет списком покупок и выводит отчёты в консоль.

## Задание (шаги)

### 1) Подготовьте структуру данных

1. Создайте массив `shoppingList`, в котором каждый элемент — объект товара.
2. Каждый товар должен иметь поля:

   * `id` (число)
   * `name` (строка)
   * `qty` (число)
   * `price` (число, цена за 1 штуку)
   * `category` (строка, например: `"food"`, `"home"`, `"tech"`)
   * `isBought` (boolean)

Пример одного элемента:

```js
{ id: 1, name: "Milk", qty: 2, price: 1.5, category: "food", isBought: false }
```

### 2) Реализуйте функции управления списком

Напишите функции:

1. `addItem(list, item)`

   * добавляет новый товар в массив
   * запрещает добавление, если `qty <= 0` или `price < 0` (выведите сообщение в консоль)

2. `removeItem(list, id)`

   * удаляет товар по `id`
   * если товара нет — выведите сообщение

3. `markAsBought(list, id)`

   * находит товар по `id`
   * меняет `isBought` на `true`

4. `updateQty(list, id, newQty)`

   * обновляет количество
   * если `newQty <= 0` — не обновляйте и выведите сообщение

### 3) Реализуйте «аналитику» (отчёты)

Напишите функции:

1. `getTotalCost(list)`

   * возвращает общую стоимость всех товаров: `sum(qty * price)`
   * используйте цикл (`for` или `while`)

2. `getUnboughtItems(list)`

   * возвращает массив товаров, у которых `isBought === false`

3. `getTotalCostByCategory(list, category)`

   * возвращает сумму стоимости товаров только выбранной категории

4. `findItem(list, predicateCallback)`

   * возвращает **первый** товар, который удовлетворяет `predicateCallback(item) === true`
   * `predicateCallback` передаётся параметром (callback-функция)

### 4) Используйте разные типы функций

Требования к стилю:

* минимум 1 **именованная** функция (function declaration)
* минимум 1 **анонимная** функция (function expression)
* минимум 1 **стрелочная** функция
* минимум 1 функция должна принимать **callback**

### 5) Добавьте объект-«сервис» с методами

Создайте объект `ShoppingService`, который хранит методы для работы со списком:

* `add`
* `remove`
* `buy`
* `total`
* `byCategory`

Внутри методов используйте **деструктуризацию** объекта товара, например:

```js
const { qty, price } = item;
```

### 6) Демонстрация работы

В конце файла:

1. Создайте стартовый список из 5–7 товаров.
2. Вызовите функции/методы в логичном порядке (добавление, удаление, покупка, отчёты).
3. Выведите в консоль результаты работы (до/после).

## Подсказки по ключевым частям

### Проверка входных данных (валидация)

* Для `qty` и `price` используйте строгие проверки:

  * `typeof qty === "number" && qty > 0`
  * `typeof price === "number" && price >= 0`

### Поиск по `id`

* Базовый подход без методов массивов (если хотите усложнить):

  * цикл `for` и сравнение `list[i].id === id`
* Либо можно использовать `for...of`, если проходите этот синтаксис.

### Callback (предикат)

* `predicateCallback` — это функция, которая возвращает `true/false`.
* Примеры:

  * найти первый товар дороже 10
  * найти первый товар категории `"tech"`

### Деструктуризация

* Используйте, когда нужно достать несколько полей из объекта:

  * `const { name, qty, price } = item;`

## Советы по улучшению работы

1. Делайте понятные сообщения в консоль: что именно произошло и почему (особенно при ошибках).
2. Не «мутируйте» данные случайно: контролируйте, где вы изменяете массив и объекты.
3. Держите функции маленькими: одна функция — одна задача.
4. Для повторяющихся операций (например, поиск по `id`) сделайте отдельную функцию `getIndexById(list, id)`.
5. Если хотите усложнить:

   * добавьте сортировку списка (купленные в конце)
   * добавьте ограничение: нельзя купить товар, которого нет
   * добавьте функцию «самый дорогой товар» через `findItem` и callback
