# Практическое задание

## Тема

**Интерактивный список дел с фильтрацией**

## Цель

Закрепить навыки работы с **JavaScript** на практике и использовать **jQuery** только как вспомогательный инструмент для подключения обработчиков событий и обращения к элементам страницы.

В этом задании вы будете тренировать:

- работу с массивами объектов;
- создание, удаление и изменение данных;
- рендеринг данных на страницу;
- обработку пользовательского ввода;
- работу с событиями;
- базовую фильтрацию списка;
- минимальное применение jQuery в реальном мини-проекте.

То есть акцент здесь именно на **логике JavaScript**, а не на большом количестве методов jQuery.

# Задание

## Что нужно сделать

Создайте небольшое веб-приложение **«Список задач»**.

Пользователь должен иметь возможность:

1. ввести текст новой задачи;
2. добавить задачу в список;
3. отметить задачу как выполненную;
4. удалить задачу;
5. отфильтровать задачи:
   - все;
   - только выполненные;
   - только невыполненные;

6. увидеть количество:
   - всех задач;
   - выполненных задач;
   - невыполненных задач.

## Техническое условие

Используйте:

- **HTML** для структуры страницы;
- **CSS** по желанию для оформления;
- **JavaScript** для всей основной логики;
- **jQuery** только для:
  - получения значений из полей ввода;
  - навешивания обработчиков событий;
  - обращения к элементам страницы;
  - при желании — очистки контейнера перед перерисовкой.

Всю основную логику хранения и обработки данных нужно реализовать именно на **чистом JavaScript**.

# Шаги

## 1. Подготовьте структуру страницы

Создайте страницу, на которой будут:

- заголовок;
- поле ввода текста задачи;
- кнопка **«Добавить»**;
- блок с кнопками фильтрации:
  - **Все**,
  - **Выполненные**,
  - **Невыполненные**;

- контейнер для списка задач;
- блок статистики.

Пример структуры:

- заголовок приложения;
- input;
- button;
- три кнопки фильтра;
- div или ul для вывода задач;
- блок для текста со статистикой.

## 2. Создайте массив для хранения задач

Каждая задача должна храниться в виде объекта.

Например, у задачи могут быть такие свойства:

- `id` — уникальный идентификатор;
- `text` — текст задачи;
- `completed` — выполнена задача или нет.

Пример структуры одного объекта:

```js
{
  id: 1,
  text: "Сделать домашнюю работу",
  completed: false
}
```

Все задачи должны храниться в одном массиве.

## 3. Реализуйте добавление задачи

Когда пользователь нажимает кнопку **«Добавить»**:

- получите текст из поля ввода;
- уберите лишние пробелы по краям;
- проверьте, что строка не пустая;
- создайте объект задачи;
- добавьте его в массив;
- очистите поле ввода;
- заново выведите список на страницу;
- обновите статистику.

## 4. Реализуйте вывод задач на страницу

Создайте отдельную функцию, например `renderTasks()`.

Она должна:

- очищать контейнер списка;
- проходить по массиву задач;
- создавать HTML для каждой задачи;
- выводить:
  - текст задачи;
  - кнопку или чекбокс для смены статуса;
  - кнопку удаления.

Если включён фильтр, выводите не все задачи, а только те, которые подходят под текущий режим отображения.

## 5. Реализуйте изменение статуса задачи

Пользователь должен иметь возможность отметить задачу как выполненную и вернуть обратно в невыполненное состояние.

Для этого:

- найдите нужную задачу по `id`;
- измените её свойство `completed`;
- заново вызовите отрисовку списка;
- обновите статистику.

## 6. Реализуйте удаление задачи

При нажатии на кнопку удаления:

- определите, какую задачу нужно удалить;
- удалите её из массива;
- заново выведите список;
- обновите статистику.

## 7. Реализуйте фильтрацию

Добавьте переменную, которая хранит текущий фильтр.

Например:

- `"all"`
- `"completed"`
- `"active"`

При нажатии на кнопки фильтрации:

- меняйте текущее значение фильтра;
- заново вызывайте функцию отрисовки.

## 8. Реализуйте блок статистики

На странице должна отображаться информация:

- всего задач;
- выполнено;
- осталось выполнить.

Статистика должна обновляться после каждого действия:

- добавления;
- удаления;
- изменения статуса;
- смены фильтра.

# Подсказки по ключевым частям

## 1. Как лучше организовать данные

Сразу приучайтесь отделять **данные** от **интерфейса**.

То есть:

- массив задач — это ваши данные;
- HTML на странице — это только отображение этих данных.

Правильный подход такой:

1. изменили массив;
2. заново вызвали функцию отрисовки;
3. страница обновилась.

Не нужно пытаться вручную менять всё по кусочкам в разных местах, если можно централизованно перерисовать интерфейс.

## 2. Как сделать уникальный `id`

Можно использовать:

- `Date.now()`
- или собственный счётчик.

Пример:

```js
const newTask = {
  id: Date.now(),
  text: taskText,
  completed: false,
};
```

## 3. Как получить задачи по текущему фильтру

Удобно сделать отдельную функцию, например `getFilteredTasks()`.

Она может возвращать:

- весь массив;
- только выполненные;
- только невыполненные.

То есть логика фильтрации должна быть отдельно, а не прямо внутри большого количества обработчиков.

## 4. Как найти задачу для изменения

Если у вас есть `id`, задачу можно найти с помощью:

- `find()` — если нужно получить сам объект;
- `findIndex()` — если нужен индекс;
- `filter()` — если нужно создать новый массив без удалённого элемента.

Это хороший повод закрепить методы массивов JavaScript.

## 5. Какую часть удобно отдать jQuery

jQuery здесь нужна минимально.

Например, удобно использовать её так:

- `$("#taskInput").val()` — получить текст из поля;
- `$("#addButton").on("click", ...)` — обработать нажатие;
- `$("#taskList").html(...)` — вставить готовую разметку;
- `$(document).on("click", ".delete-btn", ...)` — обработать клик по кнопке удаления, если элементы создаются динамически.

Но вся логика принятия решений должна оставаться в обычном JavaScript.

## 6. Как работать с динамически созданными элементами

Если кнопки удаления и переключения статуса создаются во время рендера, обычное навешивание обработчика напрямую может не сработать так, как ожидается.

Поэтому удобно использовать делегирование событий:

```js
$(document).on("click", ".delete-btn", function () {
  // логика удаления
});
```

Это важный момент, потому что элементы появляются после загрузки страницы.

# Что проверить перед отправкой (чек-лист)

Проверьте свою работу по пунктам.

### Основная логика

- задача добавляется в массив;
- пустая строка не добавляется;
- текст задачи очищается от лишних пробелов по краям;
- после добавления список обновляется;
- после добавления поле ввода очищается.

### Работа со статусом

- задачу можно отметить выполненной;
- задачу можно вернуть в невыполненное состояние;
- статус реально хранится в объекте задачи.

### Удаление

- удаляется именно выбранная задача;
- после удаления список обновляется корректно;
- другие задачи не пропадают.

### Фильтрация

- кнопка **Все** показывает все задачи;
- кнопка **Выполненные** показывает только выполненные;
- кнопка **Невыполненные** показывает только невыполненные.

### Статистика

- общее количество задач считается правильно;
- количество выполненных считается правильно;
- количество невыполненных считается правильно;
- статистика обновляется после каждого действия.

### Организация кода

- есть отдельная функция рендера;
- логика не размазана хаотично по обработчикам;
- используются массивы объектов;
- jQuery применяется минимально и по делу;
- код читается и разделён на понятные части.

# Критерии хорошей работы

Хорошая работа — это не та, где просто что-то отображается на странице. Хорошая работа — это та, где:

- логика написана аккуратно;
- данные организованы правильно;
- интерфейс обновляется предсказуемо;
- код разбит на понятные части;
- JavaScript решает задачу, а jQuery только помогает удобно связать интерфейс и логику.
